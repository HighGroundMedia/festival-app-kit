# Contributing

Thanks for your interest — contributions are welcome! This kit exists so other
volunteer-run events can reuse practical, battle-tested pieces.

## Ground rules

- **Never commit secrets or personal data.** No API keys, tokens, passwords,
  `.env` files, real attendee/volunteer/vendor data, or private URLs. Sample
  data must be tiny and obviously fake.
- **Keep modules self-contained.** Each piece should work in isolation with
  minimal dependencies.
- **Match the style** of the existing code and include a short example.

## How to contribute

1. Fork the repo and create a branch.
2. Make your change, add or update an example in `examples/`.
3. Open a pull request describing what it does and why.

Not sure about something? Open an issue and ask — happy to help.
